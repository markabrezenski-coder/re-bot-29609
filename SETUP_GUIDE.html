RE Bot — Setup Guide ~20 minutes total

This guide walks you through setting up the RE Bot to run automatically every day at 6am EST, email you results, and update your HTML tracker file — all for free.

  1. Create a GitHub account
  2. Create a new repository
  3. Upload the files
  4. Set up Gmail App Password
  5. Add secrets to GitHub
  6. Test it manually
  7. Download your updated HTML tracker

1 Create a GitHub account 2 min

  1. Go to github.com
  2. Click Sign up in the top right
  3. Enter your email, create a password, choose a username
  4. Verify your email address when prompted
GitHub is free. You don't need a paid plan — the free tier includes more than enough automation hours for a daily script.

2 Create a new repository 2 min

  1. Once logged in, click the + icon in the top right → New repository
  2. Name it: re-bot-29609
  3. Set visibility to Private (keeps your listing data private)
  4. Check Add a README file
  5. Click Create repository

3 Upload the files 5 min

You need to upload these 5 files to your repository. You can do this through the GitHub website — no coding tools needed.

Files to upload:
  • scraper.py — the main script
  • requirements.txt — Python dependencies
  • known_listings.json — starts as {}, grows over time
  • re_bot_29609.html — your HTML tracker
  • .github/workflows/re_bot_daily.yml — the scheduler

How to upload each file:

  1. In your repository, click Add file → Upload files
  2. Drag the files in (or click to browse)
  3. Click Commit changes
The workflow file needs a folder. The file re_bot_daily.yml must live inside a folder called .github/workflows/. To create it on GitHub:
  1. Click Add file → Create new file
  2. In the filename box, type: .github/workflows/re_bot_daily.yml (GitHub will auto-create the folders as you type the slashes)
  3. Copy and paste the contents of re_bot_daily.yml into the editor
  4. Click Commit new file

4 Set up Gmail App Password 5 min

Gmail requires an "App Password" for automated sending — this is a special password just for this script, separate from your normal Gmail password.

  1. Go to myaccount.google.com/security
  2. Make sure 2-Step Verification is turned on (required for App Passwords)
  3. Search for App passwords in the search bar at the top of your Google Account page
  4. Click App passwords
  5. At the bottom, enter a name like RE Bot and click Create
  6. Google will show you a 16-character password like abcd efgh ijkl mnop
  7. Copy it now — you won't see it again. Remove the spaces when you use it: abcdefghijklmnop
Keep this password safe. Anyone with it can send email from your Gmail account via this script. You can revoke it any time from the same App passwords page.

5 Add secrets to GitHub 3 min

GitHub Secrets store your credentials securely so the script can access them without exposing them in your code.

  1. In your repository, click Settings (top tab)
  2. In the left sidebar, click Secrets and variables → Actions
  3. Click New repository secret and add these two secrets:
Secret 1:
Name: GMAIL_USER
Value: your full Gmail address, e.g. yourname@gmail.com

Secret 2:
Name: GMAIL_APP_PASSWORD
Value: the 16-character app password from Step 4 (no spaces), e.g. abcdefghijklmnop

6 Test it manually 2 min

Before waiting until 6am, trigger a manual run to make sure everything works.

  1. In your repository, click the Actions tab
  2. In the left sidebar, click RE Bot Daily Scan
  3. Click Run workflow → Run workflow (green button)
  4. Wait 1–2 minutes — you'll see a yellow spinner, then a green checkmark
  5. Click the run to see the log output
  6. Check your email at markbrezenski@yahoo.com — you should receive the first report
If the run fails (red X), click into it and look at the error in the log. Common fixes:
  • Red X on "Send email": double-check your GMAIL_APP_PASSWORD secret has no spaces
  • Red X on "Commit": make sure the repository has Write permissions enabled under Settings → Actions → General → Workflow permissions → select "Read and write permissions"

7 Download your updated HTML tracker 1 min

After each run, GitHub automatically commits the updated re_bot_29609.html and known_listings.json back to your repository.

To get the latest version of your HTML tracker:

  1. Go to your repository on github.com
  2. Click re_bot_29609.html
  3. Click the download icon (↓) or Raw → then File → Save As in your browser
You're all set! The bot will now run every day at 6am EST automatically. You'll receive an email at markbrezenski@yahoo.com whether or not there are new listings — the subject line will tell you at a glance:
  • 🟡 "RE Bot · 2 NEW listing(s) found · March 18, 2026"
  • 🔵 "RE Bot · 1 price drop · March 19, 2026"
  • ⚪ "RE Bot · Daily update · March 20, 2026 · 15 active listings"

Adjusting the schedule

The schedule is set in .github/workflows/re_bot_daily.yml. The current setting is 0 11 * * * which is 11:00 UTC = 6:00 AM EST (winter) / 7:00 AM EDT (summer).

To run at 6am year-round accounting for daylight saving, you can set two schedules:

on:
  schedule:
    - cron: "0 11 * * *"  # 6am EST (Nov–Mar)
    - cron: "0 10 * * *"  # 6am EDT (Mar–Nov)

This means it runs twice on the days clocks change, but that's harmless.
